Pandragon 0.9.5: Protocol hardening, chunked file transfer, and indirect syscall support

This release includes protocol hardening fixes, migration to chunked file transfer protocol, and new indirect syscall pivot support for enhanced operational security.

- Fix RAND_B64 macro to use URL-safe base64 (RFC 4648 §5) with characters [A-Za-z0-9-_] instead of standard base64 [A-Za-z0-9+/] to avoid + and / which may be blocked or encoded differently by certain proxies
- Improve decrypt_payload error handling: return empty byte strings instead of verbose error messages, use .shape[0] for buffer length checks to work properly with memoryview objects
- Add parse_header() function for header-only parsing without full decryption (useful for debugging and initial packet inspection)
- Add new server-to-beacon opcodes: BOF_FREE, INJECT_PROCESS, MIGRATE, HOLLOW_PROCESS
- Add new beacon-to-server opcodes: LIST_FILES_RESULT
- Remove FILE_UPLOAD inline handler (replaced by chunked protocol)

- Migrate from inline file transfer to chunked file transfer protocol:
  - FILE_DOWNLOAD (inline) → FILE_DOWNLOAD_START + FILE_DOWNLOAD_CHUNK
  - FILE_UPLOAD (inline) → FILE_UPLOAD_START + FILE_UPLOAD_CHUNK
- Remove _build_file_download() and _build_file_upload() inline handlers
- Add _build_rotate_key() for ROTATE_KEY opcode (beacon ignores payload)
- Update MAX_HOLLOWING_PAYLOAD to 16MB (from previous limit)

- Add --debug CLI flag to enable verbose debug logging via PANDRAGON_DEBUG env var
- Track and log BEACON_POLL (0x02) opcode handling
- Improve malleable C2 unwrapping logic:
  - Full packet decryption using parse_packet() instead of just header parsing
  - Better debug logging throughout PSK config matching
  - Improved error handling for base64 decode failures
- Enhanced logging for empty POST bodies with headers info
- Improved registered beacon fallback with better debug output

- Add derive_beacon_id() function: derives 8-byte beacon_id from 32-byte crypto_key using SHA-256(first 8 bytes as hex) - allows configs to omit explicit beacon_id
- Auto-derive beacon_id from crypto_key if not specified in config
- Validate provided beacon_id matches derived value if both specified
- Add indirect_syscall_pivot support:
  - New config field for specifying custom Zw* function to use for indirect syscalls
  - Validate pivot starts with 'Zw' (not 'Nt')
  - Set bit 12 (0x1000) in options_bitfield when configured: indirect_pivot_set
  - Pack indirect_pivot name length + name into config blob
- Add payload_location/HTTP method mismatch warnings:
  - Warn when POST method used with query_param payload_location
  - Suggest using GET for query_param or changing to body
- Update pad_max field handling (read from config, pack as 2 bytes)


- Update test_payload_builder.py for chunked protocol changes:
  - FILE_DOWNLOAD_START passes path through unchanged
  - FILE_UPLOAD_START passes path through unchanged
  - Remove tests for inline file transfer handlers

- Fix setup.py
